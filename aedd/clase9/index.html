Búsquedas en Arreglos

En Algoritmos, la búsqueda consiste en encontrar la posición (índice) de un elemento determinado dentro de una estructura de datos.

Búsqueda Lineal

Compara secuencialmente cada elemento del arreglo desde el inicio hasta el fin.

¿Cuándo la utilizamos?
- Arreglos desordenados.
- Colecciones de tamaño pequeño.

Búsqueda Binaria

Divide el espacio de búsqueda a la mitad en cada paso aplicando "divide y vencerás".

¿Cuándo la utilizamos?
- ¡Únicamente en arreglos ordenados!
- Datos de gran volumen por su eficiencia O(log N).

Búsqueda Lineal

Recorre el arreglo elemento por elemento. Retorna el índice donde se encuentra, o -1 si no está presente.

Visualización (Buscando 7)

i = 2
22

Se compara secuencialmente:
Notas[0] (12 != 7), Notas[1] (45 != 7), Notas[2] (7 == 7) -> ¡Encontrado!

Implementación C++


int buscaElemento(int arreglo[], int tam, int elemento) {
    int pos = -1;
    bool encontrado = false;
    int i = 0;
    while (i < tam && !encontrado) {
        if (arreglo[i] == elemento) {
            pos = i;
            encontrado = true;
        }
        i++;
    }
    return pos;
}

Búsqueda Binaria

Diseñada para arreglos ordenados. En lugar de avanzar paso a paso, reduce el rango de búsqueda a la mitad comparando con el punto medio.

La idea de los Límites (Invariante):

Mantenemos dos índices de frontera:

  • a: El último elemento conocido que no cumple la condición (ej: menor que el valor buscado). Al inicio es -1.
  • b: El primer elemento conocido que sí cumple la condición (ej: mayor o igual). Al inicio es tam.

Mientras queden elementos entre ambos (b - a > 1):

Calculamos la mitad c = (a + b) / 2. Si arreglo[c] cumple, movemos la frontera b = c. Si no, movemos a = c.

Búsqueda Binaria: Visualización

Buscando el número 12 en un arreglo ordenado:

2
0
5
1
a = 1 (Menores)
8
2
c = 2 (Medio)
12
3
20
4
b = 4 (Mayores/Iguales)

En este punto, el valor en c (8) es menor que 12. Como no cumple la condición de ser ≥ 12, sabemos que la respuesta no está a la izquierda. Ajustamos la frontera inferior: a = c (o sea, a = 2).

Implementación: Búsqueda Binaria


int busquedaBinaria(int arreglo[], int tam, int elemento) {
    int a = -1;
    int b = tam;
    while (b - a > 1) {
        int c = (a + b) / 2;
        if (arreglo[c] >= elemento)
            b = c; // c cumple (es mayor o igual), achicamos por la derecha
        else
            a = c; // c no cumple (es menor), achicamos por la izquierda
    }
    // b es el primer índice que cumple (el primer elemento >= buscado)
    return b; 
}

Ejercicios Prácticos

Aplicaciones prácticas de arreglos unidimensionales

Ejercicio 1: Combinar Listas

Implementar una función que reciba dos listas ordenadas junto con sus respectivos tamaños lógicos y devuelva una nueva lista que contenga todos los elementos de ambas.

Opción A: Concatenación

Copia los elementos de la primera lista y a continuación los de la segunda. El resultado no preserva el orden global.

Opción B: Merge Combinado

Compara las cabezas de ambas listas y va insertando el menor en la tercera. Preserva el orden en tiempo óptimo O(N+M).

Solución: Ejercicio 1 (Merge Ordenado)


void mergeListas(int arr1[], int tam1, int arr2[], int tam2, int arr3[], int & tam3) {
    int i = 0, j = 0;
    tam3 = 0;
    while (i < tam1 && j < tam2) {
        if (arr1[i] < arr2[j]) {
            arr3[tam3++] = arr1[i++];
        } else {
            arr3[tam3++] = arr2[j++];
        }
    }
    while (i < tam1) arr3[tam3++] = arr1[i++]; // Copiar remanentes
    while (j < tam2) arr3[tam3++] = arr2[j++];
}

Ejercicio 2: Múltiplos de 100

Recibir una lista de enteros y su tamaño lógico. Insertar un elemento con el valor X+1 inmediatamente detrás de cada elemento X que sea múltiplo de 100. Retornar la cantidad de elementos insertados sin recorrer el arreglo más de una vez.

Estrategia eficiente O(N):

En lugar de desplazar los elementos hacia adelante repetidamente (lo cual sería O(N²)), primero contamos los múltiplos. Luego, recorremos el arreglo de atrás hacia adelante colocando cada elemento directamente en su ubicación final en un solo paso.

Solución: Ejercicio 2


int insertarMultiplos100(int arreglo[], int &tamLogico, int tamMaximo) {
    int insertados = 0;
    // 1. Contar múltiplos de 100 (distintos de cero)
    for (int i = 0; i < tamLogico; i++) {
        if (arreglo[i] != 0 && arreglo[i] % 100 == 0) insertados++;
    }
    
    int orig = tamLogico - 1;
    int dest = tamLogico + insertados - 1;
    tamLogico += insertados; // Nuevo tamaño lógico
    
    // 2. Copiar hacia atrás en una sola pasada
    while (orig >= 0) {
        if (arreglo[orig] != 0 && arreglo[orig] % 100 == 0) {
            arreglo[dest--] = arreglo[orig] + 1;
            arreglo[dest--] = arreglo[orig];
        } else {
            arreglo[dest--] = arreglo[orig];
        }
        orig--;
    }
    return insertados;
}

Ejercicio 3: Eliminar Repetidos

Recibir una lista de enteros positivos. Eliminar los valores repetidos del vector (dejando solo la primera aparición), y retornar la cantidad de números pares eliminados y el promedio de los elementos que quedaron.

Lógica del Algoritmo:

Recorremos con un índice i. Para cada elemento en V[i], verificamos si ya existía previamente en el rango [0, i-1]. Si ya existía, es repetido: si es par, sumamos uno a eliminados, desplazamos la cola del arreglo hacia la izquierda para eliminarlo, y reducimos n. Si no es repetido, avanzamos i.

Solución: Ejercicio 3


void eliminarRepetidos(int V[], int &n, int &paresEliminados, double &promedioRestantes) {
    paresEliminados = 0;
    int i = 0;
    while (i < n) {
        bool repetido = false;
        for (int j = 0; j < i; j++) {
            if (V[j] == V[i]) {
                repetido = true; break;
            }
        }
        if (repetido) {
            if (V[i] % 2 == 0) paresEliminados++;
            for (int k = i; k < n - 1; k++) {
                V[k] = V[k+1]; // Desplazar izquierda
            }
            n--; // Achicar arreglo
        } else {
            i++; // Solo avanzar si no eliminamos
        }
    }
    double suma = 0;
    for (int j = 0; j < n; j++) suma += V[j];
    promedioRestantes = (n > 0) ? (suma / n) : 0.0;
}

Ejercicio 4: Impares entre Pares

Recibir una lista ordenada de enteros positivos. Eliminar todos los valores impares que se encuentren directamente en medio de dos valores pares. Retornar la cantidad de impares eliminados y la mayor diferencia entre dos elementos pares sucesivos que queden en el vector.

Ejemplo:

Original: [2, 5, 8, 9, 12, 15, 20, 22]
El 5 está entre 2 y 8 (pares). Se elimina.
El 15 está entre 12 y 20 (pares). Se elimina.
Resultado: [2, 8, 9, 12, 20, 22]. Pares sucesivos restantes: (2,8), (8,12), (12,20), (20,22).
Mayor diferencia: 20 - 12 = 8.

Solución: Ejercicio 4


void eliminarImparesEntrePares(int V[], int &n, int &imparesEliminados, int &maxDiferencia) {
    imparesEliminados = 0;
    maxDiferencia = -1;
    int i = 1;
    // 1. Eliminar impares rodeados de pares
    while (i < n - 1) {
        if (V[i] % 2 != 0 && V[i-1] % 2 == 0 && V[i+1] % 2 == 0) {
            imparesEliminados++;
            for (int k = i; k < n - 1; k++) V[k] = V[k+1];
            n--;
        } else {
            i++;
        }
    }
    // 2. Buscar diferencia máxima entre pares sucesivos
    int ultimoParIdx = -1;
    for (int j = 0; j < n; j++) {
        if (V[j] % 2 == 0) {
            if (ultimoParIdx != -1) {
                int dif = V[j] - V[ultimoParIdx];
                if (dif > maxDiferencia) maxDiferencia = dif;
            }
            ultimoParIdx = j;
        }
    }
}
